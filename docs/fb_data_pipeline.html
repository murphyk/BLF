Pipeline: question_sets + resolution_sets → output JSON

Generated from fb_make_data.py and data_tools.py in forecast-bench-v5.

1. Loading & joining

  • Download question sets and resolution sets from GitHub (cached locally).
  • Join resolutions onto questions. For dataset questions, resolutions are looked up across all loaded date files (global lookup), not just the matching date.
  • Drop combination questions (where id is a list) for dates before a cutoff.
  • Drop unresolved questions by default (only questions with at least one resolution survive).

2. Deduplication & merging

  • Market questions appearing in multiple date files: deduped by (id, source), first occurrence kept.
  • Dataset questions: deduped by (id, source, forecast_due_date); duplicates have their resolution_dates/resolved_to lists merged.
  • Unresolved resolution dates (null values) stripped from dataset question lists.

3. Sampling (optional, via CLI flags)

  • Round-robin by source, with optional market/dataset quotas and train/test splitting.

6. Anti-leakage measures

  • Dataset resolution_criteria: original content discarded (contained direct URLs to yfinance, FRED, dbnomics, wikipedia data sources that would let the model look up the answer).
  • Dataset background (dbnomics): URLs stripped (contained exact series identifiers, e.g. db.nomics.world/meteofrance/TEMPERATURE/celsius.07747.D).
  • Market resolution_criteria: original content discarded (contained “Resolves to the outcome of the question found at {market_url}” which would let the model visit the market page).
  • market_value / market_date: only injected into the prompt when show_crowd=True, and only exists on market questions — so dataset questions can’t accidentally leak their reference value as a “market estimate.”
